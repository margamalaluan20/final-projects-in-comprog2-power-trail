# Security Policy

## Supported Versions

We currently provide security updates for the following versions of **Power-Up Run Treasure Trail**:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | ✅ Yes             |
| < 1.0   | ❌ No              |

## Reporting a Vulnerability

We take the security of our project seriously. If you discover a security vulnerability within this repository, please follow the steps below:

1. **Do not open a public issue.** Publicly disclosing a vulnerability can put other users at risk.
2. **Contact the Maintainer:** Please send an email to **[Insert Your Professional Email Here]** with the subject line `SECURITY VULNERABILITY - Power-Up Run`.
3. **Provide Details:** Include a detailed description of the vulnerability, steps to reproduce it, and the potential impact it could have on the system or user.

### Our Response Process

* **Acknowledgement:** You will receive an acknowledgement of your report within **48–72 hours**.
* **Investigation:** We will investigate the issue and may contact you for further information.
* **Resolution:** Once a fix is developed, we will push an update to the `main` branch and notify you so you can verify the solution.

## Safe Practices for Users

Since this game is built using **Python** and **Pygame**, we recommend the following to keep your environment secure:

* **Use Virtual Environments:** Always run the project within a `venv` or `conda` environment to isolate dependencies.
* **Keep Dependencies Updated:** Regularly run `pip list --outdated` and update the Pygame library to the latest stable version.
* **Verify Source:** Only download or clone this project from the official repository: [margamalaluan20/final-projects-in-comprog2-power-trail](https://github.com/margamalaluan20/final-projects-in-comprog2-power-trail).
